
# Offboarding Journey Mapping

This document maps the employee offboarding journey using a SaaS-style workflow approach, focusing on compliance, risk reduction, and structured execution.

## Journey Objective
Ensure a controlled, compliant, and auditable employee exit process while minimizing access and compliance risks.

---

## Stage 1: Exit Initiation

**Trigger:** Resignation submission or termination decision

### Key Activities
- Exit request recorded
- Exit type identified (resignation / termination)
- Notice period calculated

### Owners
- Employee (resignation)
- HR / Management (termination)

### Inputs
- Resignation letter or termination approval

### Outputs
- Exit workflow initiated

### Compliance Checkpoints
- Notice period aligned with labor requirements

---

## Stage 2: Status Update & Communication

### Key Activities
- Employee status updated to Exit Initiated
- Exit communication shared with employee and manager

### Owners
- HR

### Inputs
- Approved exit initiation

### Outputs
- Stakeholders informed

### System Touchpoints
- HRIS employee status management

---

## Stage 3: Notice Period Tracking

### Key Activities
- Notice period start and end dates tracked
- Handover plan initiated
- Ongoing compliance monitoring

### Owners
- HR
- Manager
- Employee

### Inputs
- Exit dates

### Outputs
- Notice period tracked and completed

### Compliance Checkpoints
- Attendance and obligations fulfilled

---

## Stage 4: Clearance & Handover

### Key Activities
- Asset handover completed
- Access revocation initiated
- Clearance approvals collected

### Owners
- HR
- IT
- Manager

### Inputs
- Clearance confirmations

### Outputs
- All clearances completed

### System Touchpoints
- Clearance checklist
- Approval tracking

---

## Stage 5: Final Settlement & Record Closure

### Key Activities
- Final settlement calculated
- Exit documents generated
- Employee record closed

### Owners
- HR
- Payroll

### Inputs
- Clearance confirmation
- Final attendance data

### Outputs
- Exit completed
- Employee record archived

### Success Criteria
- Compliance documentation complete
- No active access or pending liabilities

---

## Journey Design Principles
- Controlled triggers and status changes
- Clear ownership at every stage
- Audit-ready documentation
- HRIS-compatible execution
